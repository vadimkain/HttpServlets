HTTP. Servlets. 49. Servlet Filters. Часть 1

Когда из клиента передаём запрос на сервер, то всегда знаем, что есть какой-то сервлет - либо один, либо не будет такого сервлета по такому URL и получим соответствующую ошибку клиентскую, но в любом случае не может быть более одного сервлета, который обрабатывает один запрос.

В случае фильтров - это какой-то объект Apache Tomcat, который реализует интерфейс Filter и он может обрабатывать эти запросы, которые идут с клиента, до того, как они попадут в сервлет. И их может быть несколько на один и тот же URL. Мы можем использовать URL PATTERN более динамически, используя * и в таком случае мы можем несколько фильтров наложить на один и тот же запрос и что-то с этим запросом сделать. Потому что у нас все те же объекты, что и на сервлете (HttpServletRequest, HttpServletResponse). На стороне фильтров можно установить кодировку одну глобальную для всех сервлетов. Либо проверять пользователя, есть ли у него права (т.е. авторизация) и прочее.

В фильтрах также есть свой жизненный цикл. Он состоит из трёх основных частей:

  • filter.init() - вызывается один раз при первом использовании нашего фильтра.
  • filter.doFilter() - каждый раз вызывается этот метод.
  • filter.destroy() - вызывается как только завершаем работу приложения.
